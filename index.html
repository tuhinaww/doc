1. Introduction

1.1 Purpose

This document defines the functional requirements of the EcoCycle platform, a comprehensive waste categorization and sustainability platform that includes waste sorting, upcycling ideas, sustainability games, a marketplace for eco-products, and community connections. The document aims to provide clarity for stakeholders, developers, and testers involved in the development process.

1.2 Scope

EcoCycle empowers users to manage waste sustainably through advanced image recognition, educational tools, and a circular economy platform. This Functional Requirements Document (FRD) covers core modules including:

  • Waste Sorter (EcoSort)
  • Best Out of Waste
  • Eco Games
  • EcoMart Marketplace
  • EcoReuse Community
  • Integrated Chatbot

2. System Overview

EcoCycle is a comprehensive web-based platform with the following key components:

  • Image-based waste categorization using a trained machine learning model
  • Upcycling guides and sustainability education games
  • Marketplace for trading upcycled items using ERC coins
  • NGO discovery and sustainability collaboration opportunities
  • Integration with cloud and AI services

3. Functional Requirements

ID Module Requirement Description Priority
FR1 EcoSort Allow users to upload or capture waste item images for classification. High
FR2 EcoSort Process image using model to determine waste category (Recyclable, Organic, Hazardous, etc.). High
FR3 EcoSort Display disposal instructions with bin color codes based on category. High
FR4 EcoSort Support JPG, PNG, WEBP formats (Max 10MB). High
FR5 EcoSort Provide feedback for image clarity or model confidence. Medium
FR6 Best Out of Waste Provide DIY guides for converting waste into usable products. Medium
FR7 Best Out of Waste Categorize projects by difficulty and material type. Low
FR8 Eco Games Offer 4 sustainability games: Waste Sorter, Memory Game, Forest Cleaner, Bin Guessing. Medium
FR9 Eco Games Track user progress and assign Eco Badges. Low
FR10 Marketplace Allow users to upload, list, and sell upcycled products. High
FR11 Marketplace Enable ERC coin wallet management and reward system. High
FR12 Marketplace Allow users to browse and filter listed items. Medium
FR13 EcoReuse Display sustainability NGOs and their accepted material types. Medium
FR14 EcoReuse Enable users to connect with listed organizations. Medium
FR15 Auth Enable secure login and signup functionality. High
FR16 Chatbot Provide chatbot for guided navigation and support. Medium
FR17 Admin Panel Monitor user contributions, uploads, coins, and game performance. Medium

4. User Roles

Role Permissions
Guest Browse platform, play sample game, upload waste image (limited)
Registered User Access full features, upload/list in Marketplace, earn coins
Admin Manage users, oversee listings, moderate content, manage ERC coins

5. Use Case Summary

Use Case Actor Description
Upload Waste Image User User uploads image for categorization
Sort Waste Model System processes image and outputs bin color and disposal info
View DIY Guide User User explores ways to reuse waste
Play Games User User plays educational games and earns Eco Badges
List Product User User uploads a recycled product to EcoMart
Buy Product User Use ERC coins to buy upcycled items
NGO Connect User Discover and reach out to sustainability orgs
Chatbot Help User Get guided assistance via EcoBot

6. Assumptions and Dependencies

  • Internet connectivity required for model processing
  • Cloud services (AWS/GCP) to support backend operations
  • Users will provide clean, high-quality images for better model predictions
  • ERC coin economy requires proper backend logic and fraud prevention mechanisms

7. Non-Functional Requirements

Category Requirement
Performance Model inference should complete in under 100ms
Usability Platform must be mobile-responsive
Security All uploads must be scanned for malicious content
Reliability 99.5% uptime expected
Scalability Should support concurrent users and uploads efficiently
